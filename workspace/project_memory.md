## 项目长期事实

- 站点基于 GitHub Pages 提供论文阅读页。
- 论文页主要由 `src/6.generate_docs.py` 生成 Markdown front matter，再由 `app/docsify-plugin.js` 渲染。
- 当前用户希望页面默认中文优先，但保留英文原文辅助阅读与英语学习。

## 已确认口径

- 双语展示的推荐顺序：中文在前，英文保留，不做纯中文覆盖。
- “速览”必须给出真实信息，不接受泛化占位句。
- 论文页标题栏的长期口径：
  - 左侧显示英文原题；
  - 右侧显示中文标题；
  - 若只有一个标题，直接显示单栏，不保留误导性的空白半栏。
- 对于 GitHub Actions 无法访问本地 LLM 的场景，文档生成必须有可读的启发式回退，而且 TLDR、摘要、速览不能复用同一套模板句。
