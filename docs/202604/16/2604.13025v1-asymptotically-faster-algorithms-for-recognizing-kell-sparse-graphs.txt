# Asymptotically faster algorithms for recognizing ( k, ℓ )-sparse graphs 

# Bence De´ ak 1 P´ eter Madarasi 2

Abstract 

The family of ( k, ℓ )-sparse graphs, introduced by Lorea, plays a central role in combinatorial optimization and has a wide range of applications, particularly in rigidity theory. A key algorithmic problem is to decide whether a given graph is ( k, ℓ )-sparse and, if not, to produce a vertex set certifying the failure of sparsity. While pebble game algorithms have long yielded O(n2)-time recognition throughout the classical range 0 ≤ ℓ < 2k, and O(n3)-time algorithms in the extended range 2 k ≤ ℓ < 3k, substantially faster bounds were previously known only in a few special cases. We present new recognition algorithms for the parameter ranges 0 ≤ ℓ ≤ k, k < ℓ < 2k, and 2 k ≤ ℓ < 3k.Our approach combines bounded-indegree orientations, reductions to rooted arc-connectivity, augmenting-path techniques, and a divide-and-conquer method based on centroid decomposition. This yields the first subquadratic, and in fact near-linear-time, recognition algorithms throughout the classical range when instantiated with the fastest currently available subroutines. Under purely combinatorial implementations, the running times become 

O(n√n) for 0 ≤ ℓ ≤ k and O(n√n log n) for k < ℓ < 2k. For 2 k ≤ ℓ < 3k, we obtain an O(n2)-time algorithm when ℓ ≤ 2k + 1 and an O(n2 log n)-time algorithm otherwise. In each case, the algorithm can also return an explicit violating set certifying that the input graph is not ( k, ℓ )-sparse. 

# 1 Introduction 

Throughout this paper, we fix nonnegative integers k and ℓ with ℓ < 3k. In the classical range ℓ < 2k, the graph 

G = ( V, E ) is called ( k, ℓ )-sparse if, for each subset X ⊆ V , the number iG(X) of edges induced by X satisfies 

iG(X) ≤ max {k|X| − ℓ, 0}. In the extended range 2 k ≤ ℓ < 3k, we require this inequality only for subsets with size at least three. If G is ( k, ℓ )-sparse and has exactly max {k|V | − ℓ, 0} edges, then it is called ( k, ℓ )-tight . A graph is (k, ℓ )-spanning if it contains a ( k, ℓ )-tight subgraph spanning all vertices. A ( k, ℓ )-block of a ( k, ℓ )-sparse graph is a subset X ⊆ V that induces a ( k, ℓ )-tight subgraph, and a ( k, ℓ )-component is an inclusion-wise maximal ( k, ℓ )-block. These notions form the basis of a rich combinatorial theory closely tied to matroids and rigidity, and they naturally give rise to the optimization and recognition problems studied in this paper. The concept of ( k, ℓ )-sparsity originates from Lorea [1] and has since surfaced in several foundational results. In particular, the case ℓ = k characterizes exactly those graphs that decompose into k forests [2, 3]. More generally, for 

ℓ ≤ k, a graph is ( k, ℓ )-sparse precisely when its edge set can be partitioned into ℓ forests and k − ℓ pseudoforests (i.e., graphs with at most one cycle in each connected component). For ( k, ℓ ) = (2 , 3), the ( k, ℓ )-tight graphs are the Laman graphs , describing generically minimally rigid bar-joint frameworks in the plane [4], while (2 , 3)-spanning graphs correspond to rigid frameworks. In the classical range, the family of ( k, ℓ )-sparse edge sets forms the independent sets of a matroid, allowing efficient optimization over ( k, ℓ )-sparse subgraphs. In the extended range 2k ≤ ℓ < 3k, three-dimensional rigidity provides further motivation: the boundary case ℓ = 2 k plays a key role in the analysis of block–hole graphs with a single hole and yields necessary conditions for 3D rigidity when k = 3 [5]. Altogether, ( k, ℓ )-sparsity offers a unifying framework linking matroid theory, edge-disjoint spanning structures, and planar as well as spatial rigidity. In this paper, we consider the recognition problem , which asks whether a given graph is ( k, ℓ )-sparse. In the classical range, this is precisely the independence oracle for the matroid of ( k, ℓ )-sparse edge sets and thus serves as a crucial building block for numerous combinatorial algorithms, especially in rigidity theory. Throughout the paper, we treat k and ℓ as fixed constants and therefore omit poly( k, ℓ ) factors from asymptotic bounds. We denote by n

and m the numbers of vertices and edges of any input graph, respectively. In the recognition problem, we restrict attention to instances with m ≤ kn , as any graph with m > kn trivially violates the sparsity condition.

# 1.1 Historical algorithms 

For recognizing ( k, ℓ )-sparse graphs, the classical approach is the pebble game framework introduced in [6, 7, 8, 9, 10]. The pebble game is a greedy algorithm exploiting the matroidal structure of ( k, ℓ )-sparse graphs: it processes the edges one by one and inserts an edge whenever the resulting subgraph remains ( k, ℓ )-sparse. Testing whether an edge can be added is based on the Orientation Lemma [11], by maintaining a k-indegree-bounded orientation and searching for augmenting paths. In its straightforward implementation, the running time of the pebble game algorithm is O(n2). Although several faster algorithms were developed for special choices of k and ℓ, which we review below, it remained open whether recognition throughout the entire classical range could be achieved in subquadratic time. 

The lower range. For ℓ ≤ k, a graph is ( k, ℓ )-sparse if and only if its edge set can be decomposed into ℓ forests and 

k − ℓ pseudoforests [12]. This yields an incremental recognition algorithm due to Gabow and Westermann [12] that maintains such a decomposition for the accepted edges. When a new edge arrives, the algorithm first tries to insert it directly into one of the maintained parts. If this is not possible, it searches for an augmenting sequence of exchanges among the parts. The crucial point is that this search is implemented using specialized scan procedures rather than a generic matroid-union routine: the algorithm performs an initial augmentation step using the subroutine batch ,then it uses breadth-first scanning for the pseudoforest parts and cyclic scanning for the forest parts. If these scans find an augmentation, then the corresponding exchanges are performed and the edge is accepted; otherwise, the edge is rejected. This algorithm decides independence in the union of k − ℓ bicircular and ℓ graphic matroids, thereby yielding a test for ( k, ℓ )-sparsity. The same framework can also be extended to recognize ( k, k + 1)-sparse graphs — in particular, (2 , 3)-sparse graphs — by finding a maximum-size ( k, k )-sparse subgraph decomposed into k forests, and then checking whether the accepted edge set contains a nonempty top clump , i.e., an inclusion-wise maximal 

clump . In general, for ( k, ℓ )-sparse graphs, clumps are exactly the edge sets of ( k, ℓ )-tight subgraphs. For ℓ = k,once the algorithm has computed such a maximum-size ( k, k )-sparse subgraph together with a decomposition into k

forests, a top clump can be found efficiently. 

Laman graphs. For Laman graphs, a more efficient alternative to naive recognition methods is to use the notion of a red-black hierarchy , which gives an equivalent characterization: a graph is Laman if and only if it admits a red-black hierarchy. In the algorithm introduced in [13], one first performs a decomposition according to a specific set of rules and then constructs the graph corresponding to that decomposition in O(n2) time. A key observation is that the graph characterizing the decomposition is a red-black hierarchy if and only if all edges are removed during the decomposition process; therefore, instead of explicitly constructing the decomposition graph, it suffices to test whether all edges of the input graph were deleted. To compute the decomposition efficiently, the algorithm uses the fact that a graph is Laman if and only if, after duplicating any edge, the resulting multigraph can be partitioned into two edge-disjoint spanning trees. It therefore duplicates an arbitrary edge, computes such a partition, and then carries out the decomposition in O(n log n) time using tree linearization via DFS numbering together with segment trees to report and delete crossing edges. 

Outside the classical range. In the range 2 k ≤ ℓ < 3k, the standard approach is a straightforward extension of the pebble game algorithm. When deciding whether an edge uv can be inserted, one has to check, for every vertex 

w / ∈ { u, v }, whether there exists a ( k, ℓ )-tight subgraph containing all three vertices u, v, w . Performing this test separately for each choice of w leads to an O(n3)-time algorithm [14]. In the special case ℓ = 2 k, this verification can be carried out for all vertices w simultaneously using a constant number of graph searches per input edge, reducing the overall running time to O(n2) [14]. 

# 1.2 Our contribution 

We design new algorithms for all three ranges ℓ ≤ k, k < ℓ < 2k, and 2 k ≤ ℓ < 3k. In the range k < ℓ < 2k, we assume that the input graph is loop-free, as any loop would trivially violate the sparsity condition. For 2 k ≤ ℓ < 3k, we follow the standard convention and assume that the input graph is simple. A central ingredient of our approach is a novel reduction of a key subproblem of recognition to rooted arc-connectivity in directed graphs, which can be solved very efficiently for the relevant parameters. We combine this with flow-based and augmenting-path techniques and, in the most technically challenging range k < ℓ < 2k, with a divide-and-conquer method based on centroid decomposition. In each considered parameter range, we obtain significant asymptotic improvements over the best previous bounds. A notable special case is ℓ = k + 1 — and in particular the fundamental case k = 2, ℓ = 3 from combinatorial rigidity theory — for which Gabow and Westermann gave an O(n√n log n) algorithm [12]. As noted in [15], it is not clear whether this bound can be improved solely by using a faster forest decomposition procedure. The main contribution of [15] is precisely to decouple these aspects in the Laman graph case: their algorithm recognizes Laman graphs by using a forest decomposition algorithm as a black box in an initial step, thereby separating the decomposition from the subsequent computations. However, this approach is specific to Laman graphs: it does not extend to general (2 , 3)-sparse graphs, and it cannot produce a certificate of failure in the form of a vertex set violating the sparsity condition. In contrast, our algorithm works throughout the entire upper range k < ℓ < 2k, retains the decoupling 2of the forest decomposition from the later stages of the computation, and can also return a violating set when the input graph is not ( k, ℓ )-sparse.

Subroutine 1. MaxFlow( D = ( V, A ), s, t, g )

Input. A digraph D = ( V, A ) with n vertices and O(n) arcs, a source s, a sink t, and a capacity function g : A → N

such that the sum of capacities is O(n). 

Output. A pair ( x, S ), where x : A → N is a maximum g-feasible flow and s ∈ S ⊆ V \ { t} is a minimum s-t cut. 

Implementation. The best-known algorithm for the maximum flow problem [18] is based on an interior-point method, and although it has a near-linear time complexity — i.e., it is asymptotically faster than O(n1+ ε) for any 

ε > 0 — it is very difficult to use in practice. A purely combinatorial alternative is Dinic’s algorithm [19], whose running time is O(n√n) in the special case when the sum of capacities is O(n) [20].

Subroutine 3. ForestDecomp κ(G = ( V, E ))

Input. A graph G = ( V, E ) with n vertices and O(n) edges. 

Output. A tuple ( X, F 1, . . . , F κ), where X = ∅ and ( F1, . . . , F κ) is a partition of E into disjoint forests if such a partition exists; otherwise X is a vertex set that violates the ( κ, κ )-sparsity of G, and F1, . . . , F κ = ∅.

Implementation. For computing a forest decomposition, the fastest algorithm currently known [16] uses the nearly linear maximum-flow subroutine mentioned above. As a purely combinatorial alternative, one may use the Gabow-Westermann algorithm [12], which runs in time O(n√n log n). 

Note 2.1.1. It is not immediate that the Gabow–Westermann algorithm — or any other algorithm that finds κ

disjoint forests of maximum total size — can produce a violating vertex set when run on a graph that is not ( κ, κ )-sparse. Suppose that the output of such an algorithm is ( F1, . . . , F κ), and let H = ( V, F 1 ∪ · · · ∪ Fκ). It is easy to obtain a κ-indegree-bounded orientation of H: for each component of each forest Fi, choose a root and orient every edge from parent to child. With such an orientation in hand, we can then easily construct a violating set. Take any edge in E \ E(H) and attempt to insert it into H by running one iteration of the pebble game algorithm [9]. Since the insertion is impossible, this yields a certificate, namely a vertex set X that violates the ( κ, κ )-sparsity of G.

11 3.3 The range 2k ≤ ℓ < 3k

In the range 2 k ≤ ℓ < 3k, we follow the standard convention that the sparsity condition is required only for vertex sets of size at least three, and we restrict attention throughout to simple graphs. Our algorithm follows the usual pebble game paradigm. Given a graph G = ( V, E ), we construct a ( k, ℓ )-sparse subgraph H incrementally, starting from the empty graph on vertex set V and considering the edges of G one by one in an arbitrary but fixed order. For each edge uv ∈ E, we test whether adding uv to the current subgraph H preserves ( k, ℓ )-sparsity. If so, we insert uv to H; otherwise, G is not ( k, ℓ )-sparse, and the algorithm terminates. The next observation gives an exact characterization of when such an insertion is possible. 

Observation 3.3.1. Let 2 k ≤ ℓ < 3k, let H = ( V, E ) be a ( k, ℓ )-sparse graph, and let u, v ∈ V be distinct vertices with uv / ∈ E. Then the graph H′ = ( V, E ∪ { uv }) is ( k, ℓ )-sparse if and only if H contains no vertex set X ⊋ {u, v }

that violates ( k, ℓ + 1)-sparsity in H.

Proof. Since H is ( k, ℓ )-sparse, every vertex set X that does not contain both u and v continues to satisfy the sparsity bound after the edge uv is added. Therefore, if H′ is not ( k, ℓ )-sparse, then there must exist a violating vertex set 

X that contains both u and v. As the sparsity condition is imposed only on vertex sets of size at least three, such a set must satisfy X ⊋ {u, v }. Now let X ⊋ {u, v } be any vertex set. Since both u and v lie in X, we have 

iH′ (X) = iH (X) + 1 .

It follows that X violates ( k, ℓ )-sparsity in H′ if and only if it violates ( k, ℓ + 1)-sparsity in H.By the observation above and the case t = 2 of Lemma 2.2.2, deciding whether H′ is ( k, ℓ )-sparse reduces to a rooted arc-connectivity problem, provided that a k-indegree-bounded orientation of H is already available.
